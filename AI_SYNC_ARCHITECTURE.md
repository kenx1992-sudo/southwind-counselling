# 云端-本地 AI 同步架构

## 架构设计

```
┌─────────────────┐         ┌─────────────────┐
│   云端 Kimi     │ ◄────► │  本地 Szechun   │
│   (OpenClaw)    │  同步   │    (Bot)       │
└────────┬────────┘         └────────┬────────┘
         │                           │
    复杂任务处理                 用户实时对话
    • 内容生成                   • /start 回覆
    • 数据分析                   • 快速问答
    • 策略规划                   • 订单确认
```

---

## 同步机制

### 1. 消息转发
- **本地 → 云端**：复杂问题转发给 Kimi 处理
- **云端 → 本地**：处理结果通过 Szechunbot 发送给用户

### 2. 数据同步
- 共享配置文件（GitHub）
- 同步用户状态、订单信息
- 统一定价和服务内容

### 3. 分工协作
| 任务类型 | 处理方 | 示例 |
|---------|--------|------|
| 简单问答 | 本地 | "你们价格是多少？" |
| 内容生成 | 云端 | 创建社交媒体文案 |
| 复杂咨询 | 云端 | 心理健康建议 |
| 订单处理 | 本地 | 确认付款、开通服务 |

**工作流程：**

2. **你手动转发**给我（Kimi）
3. 我处理后，通过 Szechunbot 回复用户

**示例：**
```
用户 → Szechunbot: "我想了解服务"
```
